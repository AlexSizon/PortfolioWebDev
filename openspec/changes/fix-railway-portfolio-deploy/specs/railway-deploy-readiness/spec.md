## ADDED Requirements

### Requirement: Workspace lockfile supports frozen installs
The repository SHALL keep the root `pnpm-lock.yaml` synchronized with every workspace importer so Railway and other CI environments can complete `pnpm i --frozen-lockfile` successfully.

#### Scenario: Workspace dependency changes are committed with lockfile updates
- **WHEN** any workspace app changes its `dependencies` or `devDependencies`
- **THEN** the committed root lockfile MUST contain matching specifiers for that importer
- **AND** a root `pnpm i --frozen-lockfile` run MUST complete without lockfile drift errors

### Requirement: Portfolio Railway deployment uses a valid standalone entrypoint
The `portfolio` service SHALL build from the monorepo root and start from the standalone server entrypoint generated by the committed Next.js configuration.

#### Scenario: Railway deploys the portfolio app from the monorepo
- **WHEN** Railway runs the configured build and start commands for `apps/portfolio`
- **THEN** the build command MUST install workspace dependencies from the repository root and build `@repo/portfolio`
- **AND** the start command MUST reference the generated standalone `server.js` path for the portfolio app
